# Contextual Image Alt Text Generation

This feature aims to fix pages containing images with missing or unhelpful `alt` text by using AI to deduce appropriate alt text based purely on the surrounding article context, protecting user privacy and saving bandwidth since the actual images are not sent to the AI.

## User Review Required

> [!WARNING]  
> Before proceeding, please review the extraction filter logic. Is ~500 characters of surrounding text (before and after the image) sufficient for your use case, or would you prefer extracting text from the immediate parent/sibling block elements instead? 

## Proposed Changes

---

### Backend
Adds a new endpoint dedicated to bulk image alt text translation from context.

#### [MODIFY] [index.js](file:///c:/Dev/Visual_adapter/Visual_adapter_v3_backup/backend/index.js)
- Add a new HTTP handler for `POST /generate-alt`.
- Payload structure: `{ items: [ { srcUrl: "https://...", context: "...text before [CURRENT_IMAGE] text after..." } ] }`
- **Image Processing**: The Node.js backend will use `fetch()` to download the image from the `srcUrl` and convert it into a base64 inline-data format required by Gemini's multimodal API.
- **Gemini Prompt (Multimodal)**: Pass both the base64 image data AND the textual context to Gemini 2.5 Flash. Instruct Gemini: "Provide a concise but highly descriptive alt text for the exact contents of this image. Use the provided surrounding text context only as a hint for names and context, but ensure you describe what is visually happening in the photo itself."
- **Firebase Caching**: Check Firebase DB at `alt_texts/{base64_url}`. If the alt text doesn't exist, call Gemini. After generating the text, save it to Firebase under the image's source URL so it instantly applies for all future users.
- **Response**: Returns the mapping `{ "https://...": "altText" }`.

---

### Extension
Modifies the client workflow to proactively fix image tags in the cleaned article DOM.

#### [MODIFY] [content.js](file:///c:/Dev/Visual_adapter/Visual_adapter_v3_backup/extension/content.js)
- After we `tryApplyRules()` or `extractMainContent()` and get the final HTML for the reader, parse it to find `<img>` tags missing a valid `alt` attribute.
- **Heuristic Filtering (Solving the Logo/Icon Problem)**: 
  - Skip very small images (e.g., width/height < 75px).
  - Skip images with classes/src containing `icon`, `logo`, `avatar`, `sponsor`, `badge`.
  - Also, because we already isolated the `<article>` main content in a previous step, most layout icons and site logos are safely excluded already.
- **Context Extraction (Context + Image Strategy)**: 
  - For each valid image, we look at the paragraphs immediately before and after it to build context, yielding a string like `"The company announced record profits. [CURRENT_IMAGE] The graph above shows the steady climb in revenue..."`
  - Sending BOTH the image via its URL and this textual hint to the server ensures the AI knows exactly who/what it is looking at, while still objectively describing the visuals.
- Send a message `GENERATE_ALT_TEXTS` to the service worker with the aggregated contexts and image `src` URLs.
- Update the images in the DOM once the texts return.

#### [MODIFY] [background.js](file:///c:/Dev/Visual_adapter/Visual_adapter_v3_backup/extension/background.js)
- Add an `onMessage` listener for `GENERATE_ALT_TEXTS`.
- Create `handleAltTextRequest()` which executes a `POST` request against the `${BACKEND_URL}/generate-alt` endpoint.
- Returns the mapped alt strings to `content.js`, catching network errors gracefully without crashing the reader.
